# Support

## How to Get Help

If you need help with Fee Forensics, here are the best ways to get support:

## Documentation

- **README.md**: Start here for installation and usage instructions
- **CHANGELOG.md**: Check for version-specific changes and updates
- **CONTRIBUTING.md**: Guidelines for contributing to the project

## Reporting Issues

If you encounter a bug or have a feature request:

1. Check existing [GitHub Issues](https://github.com/ArttuAn/fee-forensics-agent/issues) to see if it's already been reported
2. Use the appropriate issue template (Bug Report or Feature Request)
3. Provide as much detail as possible:
   - Steps to reproduce
   - Expected vs actual behavior
   - Environment information (OS, Python version, Fee Forensics version)
   - Sample data (sanitized, no sensitive information)

## Security Issues

For security vulnerabilities, please do **not** use public issues. Instead, follow the guidelines in [SECURITY.md](SECURITY.md).

## Questions and Discussions

For general questions:
- Open a GitHub Discussion (if enabled)
- Check existing discussions for similar topics

## Contributing

We welcome contributions! Please see [CONTRIBUTING.md](CONTRIBUTING.md) for guidelines on:
- Setting up your development environment
- Running tests
- Submitting pull requests
- Code style and standards

## Response Time

We aim to respond to issues and questions within:
- **Bug reports**: 3-5 business days
- **Feature requests**: 1-2 weeks (depending on complexity)
- **Security issues**: Within 48 hours (see SECURITY.md)

## Professional Support

For enterprise support or custom development, please contact the project maintainers directly.

## Community Guidelines

Please follow our [Code of Conduct](CODE_OF_CONDUCT.md) in all interactions with the community.
