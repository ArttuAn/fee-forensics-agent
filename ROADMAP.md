# Roadmap

This document outlines the planned development direction for Fee Forensics.

## Current Status

Fee Forensics is currently at version 0.5.0 with core functionality for:
- CSV statement ingestion
- Fee and interest analysis
- Markdown and JSON report generation
- CLI interface
- Docker support
- Comprehensive testing

## Planned Features

### Version 0.6.0 (Next Release)
- [ ] Configuration file support (YAML/TOML)
- [ ] Integration tests for end-to-end workflows
- [ ] Additional sample data files
- [ ] Workflow automation scripts
- [ ] Dependabot configuration
- [ ] Enhanced documentation (ROADMAP, ARCHITECTURE)

### Version 0.7.0
- [ ] Support for multiple bank statement formats (OFX, QFX, PDF)
- [ ] Web UI for report viewing
- [ ] Scheduled audit automation
- [ ] Email notification support

### Version 0.8.0
- [ ] Machine learning for fee pattern detection
- [ ] Predictive analytics for fee forecasting

### Version 0.9.0
- [ ] Multi-language support
- [ ] Mobile app (iOS/Android)

- [ ] Collaborative features for teams

### Version 1.0.0
- [ ] Production-ready stability
- [ ] Enterprise features (SSO, RBAC)

## Long-term Vision

Fee Forensics aims to become the leading open-source tool for personal and small business financial transparency, empowering users to:
- Understand all bank fees and charges
- Negotiate better banking terms
- Make informed financial decisions
- Automate financial monitoring
- Share insights with the community
